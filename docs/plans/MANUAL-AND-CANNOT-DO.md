# Что сделать вручную и что агент не может сделать сам

После реализации плана из [roadmap-100-blocks.md](roadmap-100-blocks.md) и автоматизированных итераций ниже перечислено: **что вам нужно сделать вручную** и **что агент (Cursor/автопилот) точно не может сделать сам**.

---

## Чеклист ручных шагов (перед релизом / по необходимости)

- [ ] **Сборка:** `cd desktop && npm run tauri build`; при необходимости `unset CI`. Запуск: `npm run tauri dev`.
- [ ] **Демон:** `voiceforge daemon` в отдельном терминале; проверить подключение десктопа.
- [ ] **Keyring:** ключи `anthropic`, `openai`, `huggingface`, при необходимости `caldav_*` — см. [keyring-keys-reference.md](../runbooks/keyring-keys-reference.md).
- [ ] **Updater:** ключи подписи (`npm run tauri signer generate -w ~/.tauri/voiceforge.key`), endpoint в `tauri.conf.json`, при необходимости `TAURI_SIGNING_PRIVATE_KEY` в CI.
- [ ] **Релиз:** GitHub Release, артефакты; при желании — Flatpak/Flathub.
- [ ] **Тестирование:** E2E в среде с GUI, a11y, ручные сценарии (second instance, deep link, автозапуск).

Подробности — в разделах 1–2 ниже.

## 1. Что нужно сделать вручную (обязательные шаги)

### Сборка и запуск

- **Собрать десктоп:** `cd desktop && npm run tauri build` (на машине с установленными зависимостями; при `CI=1` может потребоваться `unset CI`). См. [desktop-build-deps.md](../runbooks/desktop-build-deps.md).
- **Запустить приложение:** `cd desktop && npm run tauri dev` — проверить работу всех новых фич (single instance, deep link, store, FTS, CalDAV виджет, updater, контекстное меню, избранное и т.д.).
- **Проверить демон:** запустить `voiceforge daemon` в отдельном терминале и убедиться, что десктоп подключается, анализ и запись работают.

### Ключи и секреты

- **Keyring:** все API-ключи (anthropic, openai, huggingface, caldav_*) должны быть сохранены в keyring (сервис `voiceforge`). Агент не имеет доступа к вашим секретам. См. [keyring-keys-reference.md](../runbooks/keyring-keys-reference.md).
- **CalDAV:** для виджета «Ближайшие встречи» нужно настроить keyring (caldav_url, caldav_username, caldav_password) или виджет покажет пустое состояние.

### Обновления (Updater)

- **Сгенерировать ключи подписи:** `cd desktop && npm run tauri signer generate -- -w ~/.tauri/voiceforge.key` (двойной дефис `--` нужен, чтобы npm не воспринимал `-w` как свой флаг). Приватный ключ хранить в безопасном месте.
- **Настроить сервер обновлений:** развернуть endpoint (или статический JSON), прописать `endpoints` и `pubkey` в `desktop/src-tauri/tauri.conf.json`. Без этого в приложении будет «Обновления отключены или недоступны». См. [desktop-updater.md](../runbooks/desktop-updater.md).
- **CI:** добавить в секреты репозитория `TAURI_SIGNING_PRIVATE_KEY` и в workflow сборки десктопа — экспорт этой переменной перед `npm run tauri build`.

### Релизы и распространение

- **Создать GitHub Release** и при необходимости загрузить артефакты (AppImage, .sig и т.д.) вручную или через CI.
- **Проверить обновление:** установить предыдущую версию, настроить endpoint, запустить приложение и нажать «Проверить сейчас».
- **Flatpak/Flathub:** подача в Flathub и модерация — ручные шаги вне репозитория.

### Тестирование, которое требует человека

- **E2E десктопа:** автоматические тесты (Playwright и т.п.) в плане (блок 84), но их настройка и прогон в среде с GUI/дисплеем — за вами.
- **Проверка доступности (a11y):** блок 87 — инструменты (axe-core и др.) можно добавить в CI, но финальная проверка с реальными пользователями и скринридерами — вручную.
- **Ручная проверка сценариев:** single instance (запуск второго экземпляра), deep link (открытие voiceforge://session/ID из браузера/терминала), автозапуск при входе в систему.

## 2. Что агент ТОЧНО не может сделать сам

- **Запуск GUI и интерактивных приложений:** агент не может открыть окно десктопа, нажать кнопки, проверить визуальное поведение. Всё, что требует «глаз и рук» на работающем приложении, — только вы.
- **Доступ к keyring и секретам:** агент не видит и не должен видеть ключи API, пароли CalDAV, приватный ключ подписи. Настройка keyring и секретов CI — только на вашей стороне.
- **Подпись артефактов и публикация релизов:** подпись обновлений (TAURI_SIGNING_PRIVATE_KEY), загрузка в Release, публикация в магазины/Flathub — действия вне кода, с доступом к секретам и аккаунтам.
- **Сетевые и внешние сервисы:** развёртывание сервера обновлений, настройка CalDAV-сервера, доступ к Jaeger/Grafana — вы делаете в своей инфраструктуре.
- **Установка системных зависимостей:** установка Node/Rust/пакетов ОС на вашей машине (в том числе в toolbox) — вы; агент может только предложить команды и документацию.
- **Решение споров и выбор продукта:** приоритизация блоков из 100, отсечение scope, принятие решений «делать ли блок 80 (заглушка Jitsi)» — за вами; агент выполняет план и предлагает порядок.

| Действие | Кто |
|----------|-----|
| Сборка/запуск десктопа, проверка в окне | Вы |

| Сервер обновлений, endpoints, Release | Вы |
| E2E и a11y «на живом приложении» | Вы |
| Код, тесты, доки, конфиг в репо | Агент |
| План 100 блоков, runbook updater, DOCS-INDEX | Агент |

После каждой сессии агент обновляет [next-iteration-focus.md](../runbooks/next-iteration-focus.md) и может выдать промпт для следующего чата. Следующие блоки из roadmap-100-blocks можно брать по приоритету и поручать агенту реализацию тех, что не требуют ручных шагов из этого списка.
