# Архив документации VoiceForge

**Назначение:** хранить устаревшие или завершённые планы, аудиты и runbook’и, чтобы не засорять актуальную документацию. Один источник правды — в корне `docs/` и `docs/runbooks/`; здесь — только история и справка.

---

## Когда переносить в архив

- **План/аудит выполнен** — основной объём задач закрыт, актуальный статус отражён в `audit/audit-to-github-map.md` и `history/closed-plans-and-roadmap.md`.
- **Документ заменён** — появился новый источник правды (например, backlog заменён на audit map).
- **Superseded** — решение или процесс заменён другим ADR/runbook’ом.

Не архивировать: текущие runbook’и, DOCS-INDEX, agent-context, next-iteration-focus, audit-to-github-map, roadmap-priority, активные ADR.

---

## Структура архива

| Каталог | Содержимое |
|---------|------------|
| `plans/` | Выполненные или устаревшие планы развития (development-plan, implementation plans). |
| `adr/` | Superseded ADR (исторические, заменённые решения). |
| `runbooks/` | Устаревшие/объёмные версии runbook'ов (например voiceforge-cursor-tz-2026). |
| `audit/` | Снимки аудита, вытесненные одним текущим документом (например PROJECT_AUDIT_AND_ROADMAP). |

---

## Как использовать

- **Для текущей работы** — смотри только актуальные документы (см. [DOCS-INDEX.md](../DOCS-INDEX.md)).
- **Для истории** — здесь лежат «как было» и завершённые планы; при необходимости можно сослаться в коммите или ADR.

---

## Правило для агента

После **большой итерации** (закрыт блок Phase или крупный план): обновить актуальные доки (DOCS-INDEX, next-iteration-focus, при необходимости runbook’и); завершённые планы и устаревшие файлы перенести сюда с кратким пояснением в коммите.
