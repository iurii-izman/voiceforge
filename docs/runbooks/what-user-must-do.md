## 2. Релиз и артефакты

| Действие | Кто | Описание |
|----------|-----|----------|
| **Подпись артефактов** | Пользователь | Подпись .deb / AppImage / updater — требует ключей и инфра; агент не подписывает. |
| **Публикация на PyPI** | Пользователь / CI | Trusted Publishing и секреты в GitHub; при пуше тега workflow может публиковать, если настроено. |
| **SHA256 для Flatpak** | Пользователь | После сборки .deb подставить реальный sha256 в `desktop/flatpak/com.voiceforge.app.yaml`. |
| **Создание GitHub Release** | CI / пользователь | При пуше тега `v*` workflow создаёт Release; черновик заметок можно править вручную. |

## 6. Краткий чеклист «что точно не сделает автопилот»

- Не разрешает KV1/KV5 и не начинает KC11/KC13 без явной формулировки «KV1 разрешён» / «KV5 разрешён».
- Не подписывает бинарники и не публикует секреты.
- Не меняет ключи в keyring и не имеет доступа к Cursor Secrets.
- Не принимает бизнес-решения (цены, упаковка, платформы).
- Не проводит живой пилот с пользователями (KV3).
- Не ставит финальный визуальный sign-off на overlay (KV2) — только документирует и готовит код.

Все остальное (код, тесты, доки, коммиты, теги, release runbook, аудит-документы) агент может делать в рамках правил репозитория и явных инструкций.
