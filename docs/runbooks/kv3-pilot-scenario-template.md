# KV3: Pilot Validation — шаблон сценария сессии

**Issue:** [#189](https://github.com/iurii-izman/voiceforge/issues/189).  
Используйте этот шаблон при проведении одной пилотной сессии с primary persona. После сессии зафиксируйте выводы в комментарии к #189 и при необходимости обновите backlog.

---

## Подготовка (5–10 мин)

- Убедиться, что демон и десктоп запущены, RAG проиндексирован (база знаний не пуста).
- Подготовить короткий сценарий «встреча»: например, 2–3 вопроса, на которые у вас есть ответы в документах (Evidence-first).
- Режим: **hybrid** или **cloud** (чтобы были карточки Answer / Do–Don't / Clarify и при необходимости deep cards).

## Ход сессии (15–25 мин)

1. **Ввод (2–3 мин)**  
   Объяснить: «Сейчас записываем только по нажатию кнопки; карточки появляются после отпускания. Ничего не уходит в облако без твоего действия (или объяснить по факту).»

2. **Проверка Evidence / Answer (5–10 мин)**  
   - Задать вопрос, ответ на который есть в загруженных документах.  
   - Оценить: карточка Answer появилась? Цитаты (Evidence) релевантны?  
   - Спросить: «Насколько это полезно в реальном звонке/встрече?»

3. **Проверка Do–Don't / Clarify (5 мин)**  
   - Задать вопрос, допускающий рекомендации (например, «как лучше сформулировать отказ?»).  
   - Оценить: Do/Don't и Clarify читаемы и по делу?

4. **Ценность и интрузивность (3–5 мин)**  
   - «Мешало ли окно с карточками? Хотелось бы больше/меньше информации?»  
   - «Готов ли использовать такое в реальных продажах/поддержке?» (или ваш сценарий.)

## После сессии

В комментарии к #189 указать:

- **Участник:** primary persona (роль/контекст без персональных данных).
- **Что прошло хорошо:** Evidence, Answer, Do–Don't, читаемость, скорость.
- **Что мешало или было непонятно:** интрузивность, формулировки, отсутствие нужных подсказок.
- **Корректировки backlog (если есть):** конкретные идеи для issues (например, «добавить тип карточки X», «уменьшить высоту overlay»).

**Completion condition для KV3:** обратная связь зафиксирована в #189; при необходимости созданы дочерние issues и обновлён program backlog.
